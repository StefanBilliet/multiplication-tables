## User Story 6 - Record difficult multiplications internally

As a parent or future version of the system,  
I want the app to remember which multiplications were answered incorrectly,  
so that weak combinations can be used later.

### Acceptance Criteria

- Given a child answers a multiplication incorrectly,
  when the answer is evaluated,
  then that multiplication is recorded as a mistake.

- Given the same multiplication is answered incorrectly multiple times,
  when each incorrect answer is evaluated,
  then the mistake record for that multiplication is updated.

- Given the child closes and reopens the app,
  when previous progress is loaded,
  then recorded mistakes remain available.
